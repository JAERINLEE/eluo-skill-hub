# Feature Specification: 대시보드 프론트엔드 구현

**Feature Branch**: `015-dashboard-frontend`
**Created**: 2026-03-04
**Status**: Draft
**Input**: User description: "대시보드 프론트엔드 구현 - 사이드바, 헤더, 메인 컨텐트 레이아웃 / 스킬 그리드 / 검색 / 페이지네이션"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 대시보드 레이아웃 및 스킬 목록 조회 (Priority: P1)

로그인한 사용자가 `/dashboard`로 진입하면, 사이드바·헤더·메인 콘텐츠 3단 레이아웃이 표시된다. 대시보드 탭이 기본 선택 상태이며, 메인 영역에 등록된 스킬들이 3열 그리드 카드 형태로 최초 9개까지 표시된다. 각 카드에는 스킬 아이콘, 제목, 설명, 카테고리 태그가 표시된다.

**Why this priority**: 대시보드의 핵심 목적인 스킬 탐색을 위한 기본 화면이며, 다른 모든 기능의 토대가 된다.

**Independent Test**: `/dashboard` 접속 시 사이드바·헤더·스킬 카드 그리드가 정상 렌더링되고, DB에 등록된 스킬 데이터가 최대 9개 표시되는지 확인하여 테스트 가능하다.

**Acceptance Scenarios**:

1. **Given** 로그인한 사용자가 있을 때, **When** `/dashboard`에 접속하면, **Then** 사이드바(대시보드·내 에이전트·카테고리), 헤더(브레드크럼), 메인 콘텐츠 영역이 표시된다.
2. **Given** 스킬이 10개 이상 등록되어 있을 때, **When** 대시보드 탭을 선택하면, **Then** 스킬 카드가 최대 9개 그리드 형태로 표시되고 "더 많은 스킬 보기" 버튼이 나타난다.
3. **Given** 스킬이 9개 이하로 등록되어 있을 때, **When** 대시보드 탭을 선택하면, **Then** 등록된 모든 스킬 카드가 표시되고 "더 많은 스킬 보기" 버튼은 표시되지 않는다.
4. **Given** 등록된 스킬이 없을 때, **When** 대시보드에 접속하면, **Then** 빈 상태 안내 메시지가 표시된다.

### User Story 2 - 스킬 검색 (Priority: P2)

사용자가 검색창에 키워드를 입력하고 "검색하기" 버튼을 클릭하거나 Enter 키를 누르면, 해당 키워드와 일치하는 스킬 목록이 필터링되어 표시된다.

**Why this priority**: 스킬이 많아질수록 원하는 스킬을 빠르게 찾는 기능이 사용성의 핵심이 된다.

**Independent Test**: 검색창에 키워드 입력 후 검색 버튼 또는 Enter 입력 시 일치하는 스킬만 표시되는지 확인하여 테스트 가능하다.

**Acceptance Scenarios**:

1. **Given** 스킬 목록이 표시된 대시보드에서, **When** 검색창에 키워드를 입력하고 "검색하기" 버튼을 클릭하면, **Then** 제목 또는 설명에 해당 키워드를 포함하는 스킬만 표시된다.
2. **Given** 검색창에 키워드를 입력한 상태에서, **When** Enter 키를 누르면, **Then** 동일하게 검색 결과가 표시된다.
3. **Given** 검색 결과가 없을 때, **When** 검색이 실행되면, **Then** "검색 결과가 없습니다" 안내 메시지가 표시된다.
4. **Given** 검색 결과가 표시된 상태에서, **When** 검색어를 지우고 다시 검색하면, **Then** 전체 스킬 목록이 다시 표시된다.

---

### User Story 3 - 추가 스킬 불러오기 (Load More) (Priority: P2)

사용자가 "더 많은 스킬 보기" 버튼을 클릭하면 다음 9개 스킬이 추가로 로드된다. 이후 페이지를 새로고침하면 지금까지 불러온 총 개수만큼의 스킬이 표시된다.

**Why this priority**: 대량의 스킬을 점진적으로 탐색할 수 있게 하는 핵심 페이지네이션 기능이다.

**Independent Test**: "더 많은 스킬 보기" 클릭 시 추가 스킬이 로드되고, 새로고침 후에도 누적 개수가 유지되는지 확인하여 테스트 가능하다.

**Acceptance Scenarios**:

1. **Given** 9개 스킬이 표시되고 추가 스킬이 존재할 때, **When** "더 많은 스킬 보기"를 클릭하면, **Then** 다음 9개 스킬이 기존 목록 아래에 추가 표시된다.
2. **Given** 18개 스킬을 불러온 상태에서, **When** 페이지를 새로고침하면, **Then** 18개의 스킬이 표시된다.
3. **Given** 모든 스킬을 불러온 상태에서, **When** 화면을 확인하면, **Then** "더 많은 스킬 보기" 버튼이 사라진다.

---

### User Story 4 - 프로필 드롭다운 및 로그아웃 (Priority: P2)

사용자가 헤더의 프로필 이미지를 클릭하면 이메일, 이름, 로그아웃 버튼이 포함된 드롭다운이 표시된다. 로그아웃 버튼을 클릭하면 세션이 종료되고 로그인 페이지로 이동한다.

**Why this priority**: 인증된 사용자의 기본적인 계정 관리 및 세션 종료 기능이다.

**Independent Test**: 프로필 이미지 클릭 시 드롭다운에 사용자 정보가 표시되고, 로그아웃 시 로그인 페이지로 이동하는지 확인하여 테스트 가능하다.

**Acceptance Scenarios**:

1. **Given** 로그인한 사용자가 대시보드에 있을 때, **When** 헤더의 프로필 이미지를 클릭하면, **Then** 사용자 이메일과 이름이 표시된 드롭다운이 나타난다.
2. **Given** 프로필 드롭다운이 열린 상태에서, **When** 로그아웃 버튼을 클릭하면, **Then** 세션이 종료되고 로그인 페이지(`/signin`)로 이동한다.
3. **Given** 프로필 드롭다운이 열린 상태에서, **When** 드롭다운 바깥 영역을 클릭하면, **Then** 드롭다운이 닫힌다.

---

### User Story 5 - 사이드바 네비게이션 및 브레드크럼 (Priority: P3)

사이드바에서 탭(대시보드, 내 에이전트, 카테고리)을 클릭하면 해당 탭이 활성화되고, 헤더의 브레드크럼이 선택된 탭에 맞게 변경된다. 카테고리 목록은 DB에서 가져온 데이터로 동적 생성된다.

**Why this priority**: 네비게이션 구조는 대시보드 확장의 기반이지만, 현재 "내 에이전트" 등의 콘텐츠는 아직 구현 대상이 아니다.

**Independent Test**: 사이드바 탭 클릭 시 활성 상태와 브레드크럼이 변경되는지 확인하여 테스트 가능하다.

**Acceptance Scenarios**:

1. **Given** 대시보드에 접속한 상태에서, **When** 사이드바의 "내 에이전트" 탭을 클릭하면, **Then** 해당 탭이 활성화되고 브레드크럼이 "ELUO AI HUB > 내 에이전트"로 변경된다.
2. **Given** 사이드바가 렌더링될 때, **When** 카테고리 섹션을 확인하면, **Then** published 스킬이 1개 이상 존재하는 카테고리만 아이콘과 함께 표시된다.
3. **Given** 카테고리 항목을 클릭했을 때, **When** 해당 카테고리를 선택하면, **Then** 브레드크럼이 "ELUO AI HUB > [카테고리명]"으로 변경되고 해당 카테고리의 스킬만 필터링되어 표시된다.

---

### User Story 6 - 스킬 북마크 및 내 에이전트 (Priority: P2)

사용자가 대시보드의 스킬 카드에서 북마크 버튼을 클릭하면 해당 스킬이 `bookmarks` 테이블에 등록되고, 사이드바의 "내 에이전트" 탭을 클릭하면 `/myagent` 페이지로 이동하여 북마크한 스킬 목록이 그리드 형태로 표시된다. `/myagent` 페이지는 대시보드와 동일한 사이드바·헤더 레이아웃을 공유한다.

**Why this priority**: 북마크는 사용자가 필요한 스킬을 개인화하여 관리하는 핵심 기능이며, "내 에이전트" 탭의 실질적인 콘텐츠를 제공한다.

**Independent Test**: 스킬 카드의 북마크 버튼 클릭 시 DB에 저장되고, `/myagent`에서 해당 스킬이 표시되는지 확인하여 테스트 가능하다.

**Acceptance Scenarios**:

1. **Given** 대시보드에서 스킬 카드가 표시된 상태에서, **When** 북마크 버튼을 클릭하면, **Then** 해당 스킬이 `bookmarks` 테이블에 등록되고 버튼이 활성화 상태로 변경된다.
2. **Given** 북마크된 스킬 카드에서, **When** 북마크 버튼을 다시 클릭하면, **Then** 북마크가 해제(삭제)되고 버튼이 비활성화 상태로 변경된다.
3. **Given** 사이드바에서, **When** "내 에이전트" 탭을 클릭하면, **Then** `/myagent` 페이지로 이동하여 북마크한 스킬이 그리드 형태로 표시된다.
4. **Given** `/myagent` 페이지에서, **When** 북마크 버튼을 클릭하여 해제하면, **Then** 해당 스킬이 목록에서 제거된다.
5. **Given** 북마크한 스킬이 없을 때, **When** `/myagent`에 접속하면, **Then** 빈 상태 안내 메시지가 표시된다.

---

### Edge Cases

- 네트워크 오류로 스킬 데이터를 불러오지 못하면 에러 메시지를 표시하고 재시도 옵션을 제공한다.
- 로그인하지 않은 사용자가 `/dashboard`에 접근하면 `/signin`으로 리다이렉트한다.
- 검색어가 공백만 포함된 경우 검색을 실행하지 않는다.
- "더 많은 스킬 보기" 클릭 중 로딩 상태를 표시하여 중복 클릭을 방지한다.
- 검색 상태에서 "더 많은 스킬 보기"를 클릭하면 검색 결과 내에서 추가 로드한다.
- 북마크 토글 중 네트워크 오류 시 이전 상태를 유지하고 에러를 표시한다.
- 동일 스킬을 중복 북마크하면 `bookmarks` 테이블의 유니크 제약 조건으로 방지한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 `/dashboard` 경로에서 사이드바, 헤더, 메인 콘텐츠로 구성된 3단 레이아웃을 표시해야 한다.
- **FR-002**: 사이드바는 "대시보드", "내 에이전트" 고정 메뉴와 DB에서 조회한 카테고리 목록을 표시해야 한다. 단, published 상태의 스킬이 1개 이상 존재하는 카테고리만 표시한다.
- **FR-003**: 헤더의 브레드크럼은 현재 선택된 사이드바 탭에 맞춰 동적으로 변경되어야 한다.
- **FR-004**: 헤더의 프로필 이미지를 클릭하면 사용자 이메일, 이름, 로그아웃 버튼이 포함된 드롭다운이 표시되어야 한다.
- **FR-005**: 로그아웃 버튼 클릭 시 사용자 세션을 종료하고 로그인 페이지로 이동해야 한다.
- **FR-006**: 대시보드 탭에서 메인 콘텐츠는 published 상태의 스킬을 3열 그리드 카드 형태로 표시해야 한다.
- **FR-007**: 각 스킬 카드에는 아이콘, 제목, 설명, 카테고리 태그를 표시해야 한다.
- **FR-008**: 스킬 데이터는 한 번에 9개씩 최신 등록 순(created_at 내림차순)으로 가져와야 한다.
- **FR-009**: 검색창에 키워드를 입력하고 "검색하기" 버튼 클릭 또는 Enter 키 입력 시 제목·설명 기준으로 매칭되는 스킬을 표시해야 한다.
- **FR-010**: "더 많은 스킬 보기" 버튼 클릭 시 다음 9개 스킬을 기존 목록에 추가해야 한다.
- **FR-011**: 추가 데이터를 불러온 후 페이지를 새로고침하면 불러온 총 개수만큼 스킬을 표시해야 한다.
- **FR-012**: 더 이상 불러올 스킬이 없으면 "더 많은 스킬 보기" 버튼을 숨겨야 한다.
- **FR-013**: 로그인하지 않은 사용자는 `/signin`으로 리다이렉트해야 한다.
- **FR-014**: 카테고리를 클릭하면 해당 카테고리에 속한 스킬만 필터링하여 표시해야 한다.
- **FR-015**: 스킬 카드는 클릭 동작 없이 목록 조회 전용이다. 스킬 상세 보기는 이 기능의 범위에 포함되지 않는다.
- **FR-016**: 각 스킬 카드에 북마크 토글 버튼이 표시되어야 한다. 북마크되지 않은 상태에서 클릭 시 `bookmarks` 테이블에 등록되고, 북마크된 상태에서 클릭 시 해제(삭제)된다.
- **FR-017**: 사이드바 "내 에이전트" 탭 클릭 시 `/myagent` 페이지로 이동하여 사용자가 북마크한 스킬을 그리드 형태로 표시해야 한다.
- **FR-018**: 대시보드 카드와 "내 에이전트" 페이지 모두에서 북마크 토글(등록/해제)이 가능해야 한다.

### Key Entities

- **Skill(스킬)**: 자동화 스킬 정보. 제목, 설명, 아이콘, 상태(published/drafted), 카테고리 관계를 포함한다.
- **Category(카테고리)**: 스킬 분류 체계. 이름과 아이콘을 가지며 사이드바 네비게이션에 동적으로 표시된다.
- **Profile(프로필)**: 사용자 정보. 이메일과 이름을 포함하며, 프로필 드롭다운에 표시된다.
- **Bookmark(북마크)**: 사용자-스킬 간 북마크 관계. `bookmarks` 테이블에 저장되며 "내 에이전트" 페이지에서 조회된다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 로그인 후 대시보드에 접속하면 3초 이내에 스킬 카드 그리드가 표시된다.
- **SC-002**: 검색어 입력 후 검색 실행 시 2초 이내에 필터링된 결과가 표시된다.
- **SC-003**: "더 많은 스킬 보기" 클릭 시 2초 이내에 추가 스킬이 로드된다.
- **SC-004**: 새로고침 후에도 이전에 불러온 스킬 수가 유지되어 사용자 경험이 단절되지 않는다.
- **SC-005**: 사용자의 90%가 원하는 스킬을 검색 또는 카테고리 필터를 통해 3번 이내의 시도로 찾을 수 있다.
- **SC-006**: 로그아웃 후 대시보드 접근 시 100% 로그인 페이지로 리다이렉트된다.

## Assumptions

- `stitch-html/dashboard.html`의 디자인(색상, 레이아웃, 카드 스타일)을 충실히 구현한다.
- 스킬 카드 그리드는 published 상태의 스킬만 표시한다 (drafted 상태 제외).
- "내 에이전트" 탭 클릭 시 `/myagent` 별도 페이지로 이동하여 북마크된 스킬을 표시한다.
- 새로고침 시 누적 스킬 개수 유지는 URL 파라미터 또는 세션 기반으로 구현한다.
- 검색은 클라이언트 사이드가 아닌 서버 사이드에서 수행하여 대량 데이터에서도 성능을 보장한다.
- 프로필 이미지는 고정 DiceBear 아바타(`https://api.dicebear.com/9.x/thumbs/svg?seed=Leo`)를 사용한다.

## Clarifications

### Session 2026-03-04

- Q: 프로필 이미지 소스는? → A: DiceBear 고정 URL (`https://api.dicebear.com/9.x/thumbs/svg?seed=Leo`) 사용
- Q: 스킬 카드 클릭 시 동작은? → A: 클릭 동작 없음 (목록 조회만, 상세 페이지는 별도 피처로 분리)
- Q: 스킬 목록 기본 정렬 순서는? → A: 최신 등록 순 (created_at 내림차순)
- Q: "내 에이전트" 탭 클릭 시 동작은? → A: "준비 중입니다" 안내 메시지 표시 (Coming Soon 플레이스홀더)

### Session 2026-03-04 (Bookmark)

- Q: "내 에이전트" 탭의 북마크 연동 방식은? → A: `/myagent` 별도 페이지로 이동하여 북마크된 스킬을 그리드로 표시
- Q: 북마크 해제(토글) 가능 여부는? → A: 대시보드 카드와 "내 에이전트" 페이지 모두에서 토글(등록/해제) 가능
- Q: "내 에이전트" 페이지 레이아웃은? → A: 대시보드와 동일한 사이드바·헤더 레이아웃 공유 (메인 콘텐츠 영역에 북마크된 스킬 그리드 표시)
