# Feature Specification: 상세설명 마크다운 렌더링 커스텀

**Feature Branch**: `017-markdown-render-custom`
**Created**: 2026-03-04
**Status**: Draft
**Input**: User description: "상세설명 마크다운 렌더링 커스텀 - 노션 컨텐츠 형식을 조사하고 노션 컨텐츠 스타일과 유사하게 CSS를 커스텀한다. 1. 메타데이터가 존재하면 메타데이터를 간략하게 보여줄 수있는 스타일을 사용한다. 2. 표 스타일은 border-radius 를 적용하고 column header에는 색상을 넣어 잘 보일 수 있게 만든다. - 표는 셀 선이 모두 보여야한다. 3. 코드블록은 터미널 CLI 형식처럼 보일 수 있도록 커스텀한다."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 표(Table) 노션 스타일 렌더링 (Priority: P1)

사용자가 스킬 상세 팝업에서 마크다운 콘텐츠에 포함된 표를 볼 때, 노션처럼 깔끔하고 구조화된 표 스타일로 렌더링된다. 표의 모서리는 둥글게 처리되고, 컬럼 헤더에는 배경색이 적용되어 한눈에 구분 가능하며, 모든 셀 경계선이 명확하게 표시된다.

**Why this priority**: 표는 스킬 상세 가이드에서 단축키 목록, 파라미터 설명 등 핵심 정보를 전달하는 가장 빈번한 요소이므로 최우선으로 개선해야 한다.

**Independent Test**: 마크다운 콘텐츠에 GFM 표를 포함한 스킬을 열어 표가 border-radius, 헤더 색상, 셀 경계선이 모두 적용된 상태로 렌더링되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 마크다운에 GFM 표가 포함된 스킬이 존재할 때, **When** 사용자가 스킬 상세 팝업을 열면, **Then** 표 외곽에 border-radius가 적용되어 둥근 모서리로 표시된다.
2. **Given** 마크다운에 GFM 표가 포함된 스킬이 존재할 때, **When** 사용자가 스킬 상세 팝업을 열면, **Then** 표의 첫 번째 행(컬럼 헤더)에 구분 가능한 배경색이 적용된다.
3. **Given** 마크다운에 GFM 표가 포함된 스킬이 존재할 때, **When** 사용자가 스킬 상세 팝업을 열면, **Then** 모든 셀(th, td)에 경계선이 보인다.
4. **Given** 표의 내용이 컨테이너 폭을 초과할 때, **When** 사용자가 해당 표를 볼 때, **Then** 가로 스크롤이 가능하며 레이아웃이 깨지지 않는다.

---

### User Story 2 - 코드블록 터미널 CLI 스타일 렌더링 (Priority: P1)

사용자가 스킬 상세 가이드에서 코드블록을 볼 때, 터미널/CLI 형태의 시각적 스타일로 표시된다. 어두운 배경에 밝은 텍스트, 상단에 파일명 또는 언어 표시 바가 있어 실제 터미널을 보는 듯한 느낌을 준다.

**Why this priority**: 코드블록은 스킬 실행 방법, 설정 예시 등 사용자가 복사하여 바로 사용할 핵심 콘텐츠이므로 가독성이 매우 중요하다.

**Independent Test**: 마크다운에 코드블록(````typescript` 등)을 포함한 스킬을 열어 어두운 배경의 터미널 스타일 코드블록이 렌더링되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 마크다운에 펜스 코드블록이 포함된 스킬이 존재할 때, **When** 사용자가 스킬 상세 팝업을 열면, **Then** 코드블록이 어두운 배경(다크 테마)에 밝은 텍스트로 표시된다.
2. **Given** 코드블록에 언어가 지정되어 있을 때(예: ```typescript), **When** 해당 블록이 렌더링되면, **Then** 코드블록 상단에 언어명이 표시되는 헤더 바가 나타난다.
3. **Given** 코드블록의 내용이 길 때, **When** 사용자가 해당 블록을 볼 때, **Then** 가로 스크롤이 가능하며 구문 하이라이팅이 적용된다.
4. **Given** 인라인 코드(백틱 1개)가 사용될 때, **When** 해당 텍스트가 렌더링되면, **Then** 인라인 코드는 배경색과 구분되는 스타일로 표시되되 블록 코드와는 다른 스타일이 적용된다.

### User Story 3 - 메타데이터 영역 스타일링 (Priority: P2)

스킬 마크다운 콘텐츠의 최상단에 YAML 프론트매터(frontmatter) 형식의 메타데이터가 존재하는 경우, 이를 파싱하여 별도의 정보 카드 형태로 간략하게 표시한다. 메타데이터가 없으면 아무것도 표시하지 않는다.

**Why this priority**: 메타데이터는 스킬의 버전, 호환성, 의존성 등 부가 정보를 전달하는 선택적 요소이므로, 핵심 렌더링(표/코드블록) 이후에 처리한다.

**Independent Test**: YAML 프론트매터가 포함된 마크다운 콘텐츠를 가진 스킬을 열어 메타데이터가 정보 카드 형태로 표시되는지, 프론트매터 없는 콘텐츠에서는 메타데이터 영역이 표시되지 않는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 마크다운 콘텐츠 최상단에 YAML 프론트매터(`---`로 감싼 영역)가 존재할 때, **When** 사용자가 스킬 상세 팝업을 열면, **Then** 메타데이터가 key-value 형태의 간략한 정보 카드로 표시된다.
2. **Given** 마크다운 콘텐츠에 프론트매터가 없을 때, **When** 사용자가 스킬 상세 팝업을 열면, **Then** 메타데이터 영역이 표시되지 않는다.
3. **Given** 메타데이터 카드가 표시될 때, **When** 사용자가 해당 영역을 볼 때, **Then** 본문 콘텐츠와 시각적으로 구분되는 스타일(배경색, 테두리 등)이 적용된다.

---

### User Story 4 - 노션 스타일 기본 타이포그래피 (Priority: P2)

마크다운 콘텐츠의 제목(h1~h6), 본문, 목록, 인용문 등 기본 요소들이 노션과 유사한 깔끔한 타이포그래피 스타일로 렌더링된다.

**Why this priority**: 기본 타이포그래피는 전체적인 콘텐츠 가독성을 결정하지만, 핵심 요소(표/코드블록)보다는 범용적인 개선이므로 P2로 설정한다.

**Independent Test**: 다양한 마크다운 요소(제목, 목록, 인용문, 구분선, 링크)가 포함된 콘텐츠를 열어 노션과 유사한 스타일로 렌더링되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 마크다운에 h1~h3 제목이 포함될 때, **When** 렌더링되면, **Then** 각 레벨별로 크기와 두께가 구분되며, 노션처럼 제목 하단에 적절한 여백이 적용된다.
2. **Given** 마크다운에 순서 있는/없는 목록이 포함될 때, **When** 렌더링되면, **Then** 들여쓰기와 마커 스타일이 깔끔하게 적용된다.
3. **Given** 마크다운에 인용문(blockquote)이 포함될 때, **When** 렌더링되면, **Then** 왼쪽에 세로 바와 함께 배경색이 살짝 적용되어 본문과 구분된다.

---

### Edge Cases

- 마크다운 콘텐츠가 비어있거나 null일 때: 기존 빈 상태 UI 그대로 유지
- 표가 하나의 컬럼만 존재할 때: 정상 렌더링되며 border-radius 유지
- 코드블록에 언어가 지정되지 않았을 때: 언어 헤더 바를 표시하지 않거나 "Code"로 기본 표시
- YAML 프론트매터의 값이 매우 길 때: 텍스트 잘림 처리(ellipsis)
- 프론트매터 형식이 올바르지 않을 때: 파싱 실패 시 메타데이터 영역을 숨기고 원본 텍스트를 본문으로 표시
- 중첩된 표(표 안의 표)가 있을 때: 기본 렌더링 유지, 스타일 깨짐 방지
- 매우 긴 코드라인이 있을 때: 가로 스크롤 적용, 줄바꿈 없음
- 다크모드 지원: 현재 라이트 모드만 지원하므로 다크모드는 범위 밖

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 GFM 표를 렌더링할 때 표 외곽에 border-radius를 적용하여 둥근 모서리로 표시해야 한다.
- **FR-002**: 시스템은 표의 첫 번째 행(thead)에 배경색을 적용하여 컬럼 헤더를 시각적으로 구분해야 한다.
- **FR-003**: 시스템은 표의 모든 셀(th, td)에 경계선(border)을 표시하여 셀 구분이 명확해야 한다.
- **FR-004**: 시스템은 펜스 코드블록을 어두운 배경 + 밝은 텍스트의 터미널 스타일로 렌더링해야 한다.
- **FR-005**: 시스템은 언어가 지정된 코드블록에 대해 상단 헤더 바에 언어명을 표시해야 한다.
- **FR-006**: 시스템은 코드블록에 구문 하이라이팅(syntax highlighting)을 적용해야 한다.
- **FR-007**: 시스템은 마크다운 콘텐츠 최상단의 YAML 프론트매터를 감지하고, 존재 시 key-value 정보 카드로 렌더링해야 한다.
- **FR-008**: 시스템은 프론트매터가 없는 콘텐츠에서는 메타데이터 영역을 표시하지 않아야 한다.
- **FR-009**: 시스템은 인라인 코드를 블록 코드와 시각적으로 구분되는 별도 스타일로 렌더링해야 한다.
- **FR-010**: 시스템은 제목(h1~h3), 목록, 인용문 등 기본 마크다운 요소에 노션과 유사한 타이포그래피 스타일을 적용해야 한다.
- **FR-011**: 시스템은 표와 코드블록이 컨테이너 폭을 초과할 때 가로 스크롤을 제공해야 한다.
- **FR-012**: 시스템은 기존 스킬 관리(admin) 마크다운 미리보기에 영향을 주지 않고, 스킬 상세 팝업 전용 스타일을 적용해야 한다.

### Key Entities

- **MarkdownContent**: 스킬의 markdown_content 필드에 저장된 마크다운 텍스트. 선택적으로 YAML 프론트매터를 포함할 수 있음.
- **Frontmatter Metadata**: 마크다운 최상단 `---` 블록으로 감싸진 YAML 데이터. key-value 쌍으로 구성되며, 버전/호환성/태그 등 부가 정보를 포함.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 표가 포함된 스킬 상세 팝업을 열었을 때, 표의 모서리가 둥글게 표시되고, 헤더 행에 배경색이 적용되며, 모든 셀 경계선이 보인다.
- **SC-002**: 코드블록이 포함된 스킬 상세 팝업을 열었을 때, 어두운 배경의 터미널 스타일 코드블록으로 렌더링되고 구문 하이라이팅이 적용된다.
- **SC-003**: YAML 프론트매터가 있는 콘텐츠를 열면 메타데이터 정보 카드가 표시되고, 없는 콘텐츠에서는 표시되지 않는다.
- **SC-004**: 기존 관리자 스킬 등록/수정 화면의 마크다운 미리보기 스타일은 변경되지 않는다.
- **SC-005**: 모든 마크다운 요소(제목, 목록, 인용문, 링크, 이미지)가 시각적으로 노션 스타일과 유사하게 렌더링된다.

## Assumptions

- 마크다운 콘텐츠는 기존 `react-markdown` + `remark-gfm` 기반으로 렌더링되며, 이번 변경은 CSS/컴포넌트 커스텀에 집중한다.
- YAML 프론트매터 파싱은 클라이언트 라이브러리 또는 간단한 정규식으로 처리할 수 있다.
- 노션 스타일의 "유사함"은 색상 팔레트, 여백, 타이포그래피의 전반적 느낌을 의미하며, 픽셀 단위의 완벽한 복제를 의미하지 않는다.
- 기존 admin MarkdownPreview 컴포넌트와 별도로, 스킬 상세 팝업 전용 마크다운 렌더러를 만들거나 기존 컴포넌트를 확장하여 사용한다.
- 코드블록 구문 하이라이팅 다크 테마는 highlight.js 다크 테마 CSS를 활용한다.
- 프론트매터의 key 이름은 자유 형식이며, 표준화된 스키마는 이번 범위에 포함하지 않는다.
