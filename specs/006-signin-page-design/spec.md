# Feature Specification: 로그인 페이지 디자인 리뉴얼

**Feature Branch**: `006-signin-page-design`
**Created**: 2026-03-03
**Status**: Draft
**Input**: User description: "로그인페이지 디자인 - signin.html 참고 디자인 적용, 로고 교체, eluo 폰트 적용, BackgroundBeamsWithCollision 배경 효과 추가"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 로그인 페이지 시각적 리뉴얼 (Priority: P1)

사용자가 로그인 페이지에 접속하면 브랜드 아이덴티티가 반영된 디자인(글래스모피즘 카드, 브랜드 컬러, eluo 로고)을 볼 수 있다. 기존 기능(이메일/비밀번호 입력, 로그인 버튼, 회원가입 링크)은 동일하게 유지되며, 시각적 완성도만 향상된다.

**Why this priority**: 로그인 페이지는 사용자가 서비스를 처음 접하는 핵심 접점이므로, 브랜드 신뢰감을 높이기 위해 가장 중요하다.

**Independent Test**: 로그인 페이지 URL 접속 시 signin.html 레퍼런스와 일치하는 레이아웃/스타일이 렌더링되는지 시각적으로 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 사용자가 로그인 페이지에 접속했을 때, **When** 페이지가 로드되면, **Then** 글래스모피즘 스타일의 로그인 카드가 화면 중앙에 표시된다.
2. **Given** 사용자가 로그인 페이지에 접속했을 때, **When** 페이지가 로드되면, **Then** 상단 헤더에 eluo-logo.svg 로고와 "ELUO AI SKILL HUB" 타이틀이 표시된다.
3. **Given** 사용자가 로그인 페이지에 접속했을 때, **When** 페이지가 로드되면, **Then** 헤더 타이틀은 eluo 커스텀 폰트(font-eluo)로 렌더링된다.
4. **Given** 사용자가 로그인 폼에 이메일과 비밀번호를 입력하고 로그인 버튼을 누르면, **When** 인증이 성공하면, **Then** 대시보드로 리다이렉트된다 (기존 기능 유지).

---

### User Story 2 - 배경 애니메이션 효과 (Priority: P2)

사용자가 로그인 페이지에 접속하면 BackgroundBeamsWithCollision 컴포넌트가 배경에 표시되어, 빛줄기가 하강하며 하단 경계와 충돌 시 파티클 폭발 효과를 보여준다. 이 애니메이션은 페이지의 시각적 매력을 높이되, 로그인 폼 사용에 방해가 되지 않아야 한다.

**Why this priority**: 배경 효과는 시각적 차별화 요소이지만, 핵심 로그인 기능과 기본 디자인이 먼저 완성되어야 한다.

**Independent Test**: 로그인 페이지 로드 시 배경에 빛줄기 애니메이션이 반복 재생되고, 하단 경계에 도달하면 폭발 효과가 나타나는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 사용자가 로그인 페이지에 접속했을 때, **When** 페이지가 로드되면, **Then** 배경에 여러 개의 빛줄기가 위에서 아래로 이동하는 애니메이션이 표시된다.
2. **Given** 빛줄기가 하단 경계에 도달했을 때, **When** 충돌이 감지되면, **Then** 충돌 지점에서 파티클 폭발 효과가 나타난다.
3. **Given** 배경 애니메이션이 재생 중일 때, **When** 사용자가 로그인 폼을 사용하면, **Then** 애니메이션이 폼 조작을 방해하지 않는다 (폼은 배경보다 위에 위치).

---

### User Story 3 - 반응형 디자인 (Priority: P2)

모바일부터 데스크톱까지 다양한 화면 크기에서 로그인 페이지가 적절하게 표시된다. 로그인 카드 너비, 패딩, 헤더 레이아웃이 화면 크기에 따라 조정된다.

**Why this priority**: 다양한 디바이스에서 접속하는 사용자 경험을 보장하기 위해 중요하다.

**Independent Test**: 모바일(375px), 태블릿(768px), 데스크톱(1280px) 뷰포트에서 페이지가 깨지지 않고 적절히 표시되는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 모바일 디바이스(375px 이하)에서 접속했을 때, **When** 페이지가 로드되면, **Then** 로그인 카드가 좌우 여백을 유지하며 전체 너비에 맞게 표시된다.
2. **Given** 데스크톱(1280px 이상)에서 접속했을 때, **When** 페이지가 로드되면, **Then** 로그인 카드 최대 너비가 440px로 제한되고 화면 중앙에 위치한다.

---

### Edge Cases

- 사용자의 브라우저가 backdrop-filter(글래스모피즘)를 지원하지 않을 때 적절한 폴백 배경이 표시되는가?
- framer-motion이 로드되지 않거나 JS가 비활성화된 환경에서 로그인 폼은 정상적으로 사용 가능한가?
- 매우 긴 이메일 주소를 입력했을 때 입력 필드 레이아웃이 깨지지 않는가?
- 비밀번호 표시/숨기기 토글 버튼이 정상 동작하는가?

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 로그인 페이지 상단에 헤더가 표시되어야 하며, 좌측에 eluo-logo.svg 로고와 "ELUO AI SKILL HUB" 타이틀, 우측에 로그인 버튼이 포함되어야 한다.
- **FR-002**: 헤더 타이틀("ELUO AI SKILL HUB")은 프로젝트에 설정된 eluo 커스텀 폰트(font-eluo)로 렌더링되어야 한다.
- **FR-003**: 로그인 카드는 글래스모피즘 스타일(반투명 배경 70% 불투명도, blur 12px, 미세 테두리)로 표시되어야 한다.
- **FR-004**: 로그인 카드 상단에 브랜드 아이콘(노란색 배경 라운드 사각형 내 아이콘), "로그인" 타이틀, "ELUO AI Skill Hub에 접속하세요" 서브텍스트가 표시되어야 한다.
- **FR-005**: 이메일 입력 필드(placeholder: "example@eluocnc.com"), 비밀번호 입력 필드(표시/숨기기 토글 포함), 비밀번호 찾기 링크, 로그인 버튼이 포함되어야 한다.
- **FR-006**: 로그인 카드 하단에 구분선과 "계정이 없으신가요?" 텍스트, 회원가입 링크가 표시되어야 한다.
- **FR-007**: 배경에 BackgroundBeamsWithCollision 컴포넌트가 렌더링되어야 하며, 빛줄기 하강 및 충돌 파티클 애니메이션이 반복 재생되어야 한다.
- **FR-008**: 배경 색상은 브랜드 컬러(brand-yellow, brand-navy)의 방사형 그라디언트를 적용해야 한다.
- **FR-009**: 기존 로그인 인증 기능(이메일/비밀번호 검증, 서버 사이드 인증, 대시보드 리다이렉트)은 변경 없이 유지되어야 한다.
- **FR-010**: 비밀번호 필드에 표시/숨기기 토글 아이콘(눈 아이콘)이 제공되어야 한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 로그인 페이지가 signin.html 레퍼런스 디자인과 시각적으로 90% 이상 일치한다 (레이아웃, 색상, 타이포그래피, 간격).
- **SC-002**: 사용자가 로그인 과정을 15초 이내에 완료할 수 있다 (디자인 변경이 사용성을 저하시키지 않음).
- **SC-003**: 모바일(375px)~데스크톱(1920px) 뷰포트에서 레이아웃 깨짐 없이 정상 표시된다.
- **SC-004**: 배경 애니메이션이 부드럽게 재생된다 (시각적으로 끊김이 느껴지지 않음).
- **SC-005**: 페이지 첫 콘텐츠 표시가 2초 이내에 이루어진다.

## Assumptions

- `/public/eluo-logo.svg` 파일은 사용자가 수동으로 public 폴더에 배치할 예정이다.
- 기존 eluo 폰트(ELUOFACEVF.ttf)와 Pretendard 폰트가 프로젝트에 이미 설정되어 있다.
- 브랜드 컬러(brand-yellow: #FEFE01, brand-navy: #00007F, brand-light: #F0F0F0)는 globals.css에 이미 정의되어 있다.
- shadcn UI 컴포넌트(Button, Input, Label, Card)가 이미 프로젝트에 설치되어 있다.
- framer-motion 패키지를 신규 설치해야 한다.
- 비밀번호 찾기 기능은 이번 피처 범위에 포함되지 않으며, 링크만 표시한다.
- 헤더 컴포넌트는 로그인 페이지 전용으로 작성하거나, 공용 컴포넌트로 분리할 수 있다.

## Scope

### In Scope

- 로그인 페이지 UI/UX 디자인 리뉴얼 (signin.html 기반)
- 헤더 컴포넌트 추가 (로고 + 타이틀 + 로그인 버튼)
- 글래스모피즘 스타일 로그인 카드
- BackgroundBeamsWithCollision 배경 효과 컴포넌트 통합
- 비밀번호 표시/숨기기 토글
- framer-motion 패키지 설치
- 반응형 디자인

### Out of Scope

- 로그인 인증 로직 변경
- 비밀번호 찾기 기능 구현
- 회원가입 페이지 디자인 (별도 피처)
- 다크 모드 지원
- 접근성(a11y) 전면 감사
