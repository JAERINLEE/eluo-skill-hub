## User Scenarios & Testing *(mandatory)*

### User Story 1 - 탭 종료 시 자동 로그아웃 (Priority: P1)

사용자가 로그인한 상태에서 브라우저 탭을 닫으면 세션이 즉시 제거되어, 새 탭에서 사이트에 접속하면 로그인 페이지가 표시된다. 탭 하나라도 닫으면 로그아웃되어야 한다.

**Why this priority**: 현재 탭/브라우저를 닫아도 세션이 유지되어 보안 취약점이 존재하며, 이를 해결하는 것이 핵심 목표이다.

**Independent Test**: 로그인 후 탭을 닫고, 새 탭에서 사이트에 접속하여 로그인 페이지로 리다이렉트되는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 사용자가 로그인한 상태, **When** 현재 탭을 닫고 새 탭에서 사이트에 접속, **Then** 로그아웃 상태이며 로그인 페이지가 표시된다
2. **Given** 사용자가 로그인한 상태, **When** 브라우저를 완전히 종료 후 다시 실행하여 사이트에 접속, **Then** 로그아웃 상태이며 로그인 페이지가 표시된다
3. **Given** 사용자가 로그인한 상태, **When** 명시적으로 로그아웃 버튼을 클릭, **Then** 즉시 세션이 제거되고 로그인 페이지로 이동한다

---

### Edge Cases

- 브라우저의 "이전 세션 복원" 기능이 활성화된 경우에도 세션이 만료되어야 한다
- 여러 탭에서 동시에 로그인한 상태에서 한 탭을 닫으면, 해당 탭의 세션만 제거되고 나머지 열린 탭의 세션은 유지된다
- 세션 토큰이 완전히 만료된 경우 자동으로 로그인 페이지로 리다이렉트되어야 한다
- 탭 종료 감지가 실패한 경우(브라우저 크래시 등)에도 세션 쿠키의 만료로 보완되어야 한다

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 탭 종료(닫기) 시 세션을 제거하여 자동 로그아웃 처리해야 한다
- **FR-002**: 시스템은 탭 종료 감지를 위해 클라이언트 사이드 이벤트 핸들러를 사용해야 한다
- **FR-003**: 시스템은 인증 세션 쿠키를 세션 쿠키(session cookie, maxAge/expires 없음)로 설정하여 브라우저 종료 시에도 자동 삭제되도록 해야 한다
- **FR-004**: 사용자가 명시적으로 로그아웃한 경우 즉시 세션이 제거되어야 한다
- **FR-005**: 세션 만료 후 보호된 페이지 접근 시 로그인 페이지로 리다이렉트해야 한다

## Clarifications

### Session 2026-03-06

- Q: "브라우저 종료"의 범위 — 탭 닫기와 브라우저 전체 종료 중 어디까지 로그아웃 대상인가? → A: 탭 하나라도 닫으면 로그아웃
- Q: 여러 탭이 열려 있을 때, 한 탭을 닫으면 나머지 열린 탭도 로그아웃해야 하는가? → A: 닫은 탭만 로그아웃, 나머지 열린 탭은 유지

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 탭을 닫고 새 탭에서 접속 시 100% 로그아웃 상태로 전환된다
- **SC-002**: 브라우저를 완전히 종료 후 재접속 시 100% 로그아웃 상태로 전환된다
- **SC-003**: 명시적 로그아웃 시 즉시 세션이 제거된다

## Assumptions

- 현재 Supabase Auth의 `@supabase/ssr`이 영속적 쿠키(persistent cookie)로 세션을 저장하여 브라우저 종료 후에도 세션이 유지되는 것이 원인이다
- 탭 종료 감지는 `beforeunload` 또는 `visibilitychange` 이벤트를 활용하여 로그아웃 API를 호출하는 방식으로 구현한다
- 세션 쿠키(maxAge/expires 없음) 전환으로 브라우저 종료 시의 보완 메커니즘을 제공한다
- 공유 PC/키오스크 등의 보안 환경을 고려한 정책이다
