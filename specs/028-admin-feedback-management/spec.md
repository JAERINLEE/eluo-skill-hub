# Feature Specification: 어드민 피드백 관리 페이지

**Feature Branch**: `028-admin-feedback-management`
**Created**: 2026-03-10
**Status**: Draft
**Input**: User description: "어드민 피드백관리 페이지 구현 - 스킬에 작성된 피드백을 테이블뷰로 보여주고, 아코디언으로 댓글 확인/작성, 비밀글 연동, 페이지네이션 적용"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 피드백 목록 테이블뷰 조회 (Priority: P1)

관리자가 어드민 피드백 관리 페이지에 접근하면, 모든 스킬에 대해 작성된 피드백을 테이블 형태로 확인할 수 있다. 테이블에는 피드백 작성자, 스킬명, 평점, 코멘트 요약, 비밀글 여부, 작성일, 댓글 수가 표시된다. 피드백 목록은 최신순으로 정렬되며, 페이지네이션이 적용된다.

**Why this priority**: 피드백 관리의 핵심 기능으로, 관리자가 사용자 피드백을 한눈에 파악하기 위한 기반 기능

**Independent Test**: 관리자가 /admin/feedbacks 페이지에 접근하여 피드백 목록이 테이블 형태로 렌더링되는지 확인

**Acceptance Scenarios**:

1. **Given** 피드백이 존재하는 상태에서, **When** 관리자가 피드백 관리 페이지에 접근하면, **Then** 피드백 목록이 테이블 형태로 최신순 정렬되어 표시된다
2. **Given** 피드백이 10개를 초과하는 상태에서, **When** 관리자가 다음 페이지 버튼을 클릭하면, **Then** 다음 페이지의 피드백 목록이 표시된다
3. **Given** 비밀글 피드백이 존재할 때, **When** 관리자가 테이블을 확인하면, **Then** 비밀글 피드백도 내용이 표시되며 비밀글 아이콘/뱃지로 구분된다
4. **Given** 피드백이 없는 상태에서, **When** 관리자가 페이지에 접근하면, **Then** "등록된 피드백이 없습니다" 빈 상태 메시지가 표시된다

---

### User Story 2 - 아코디언으로 댓글 조회 (Priority: P1)

관리자가 피드백 행을 클릭하면 아코디언이 펼쳐지며, 해당 피드백에 달린 댓글 목록을 확인할 수 있다. 댓글은 최신순으로 표시된다.

**Why this priority**: 피드백에 대한 기존 응답 이력을 확인하는 것은 댓글 작성 전 필수 과정

**Independent Test**: 댓글이 존재하는 피드백 행을 클릭하여 아코디언이 열리고 댓글 목록이 표시되는지 확인

**Acceptance Scenarios**:

1. **Given** 댓글이 달린 피드백이 있을 때, **When** 관리자가 해당 피드백 행을 클릭하면, **Then** 아코디언이 펼쳐지며 댓글이 최신순으로 표시된다
2. **Given** 댓글이 없는 피드백이 있을 때, **When** 관리자가 해당 피드백 행을 클릭하면, **Then** 아코디언이 펼쳐지며 "댓글이 없습니다" 메시지와 댓글 작성 폼이 표시된다
3. **Given** 아코디언이 열린 상태에서, **When** 관리자가 같은 행을 다시 클릭하면, **Then** 아코디언이 접힌다

---

### User Story 3 - 관리자 댓글 작성 (Priority: P1)

관리자가 아코디언을 펼친 상태에서 댓글 입력 폼을 통해 새 댓글을 작성하고 등록할 수 있다. 등록 후 댓글 목록이 즉시 갱신된다.

**Why this priority**: 사용자 피드백에 대한 관리자 응답은 피드백 관리의 핵심 목적

**Independent Test**: 아코디언 내 댓글 작성 폼에 내용을 입력하고 등록 버튼을 눌러 댓글이 추가되는지 확인

**Acceptance Scenarios**:

1. **Given** 아코디언이 열린 상태에서, **When** 관리자가 댓글을 입력하고 등록 버튼을 누르면, **Then** 댓글이 저장되고 목록에 즉시 반영된다
2. **Given** 아코디언이 열린 상태에서, **When** 관리자가 빈 내용으로 등록 버튼을 누르면, **Then** 등록되지 않고 유효성 검증 메시지가 표시된다
3. **Given** 댓글 등록 중 네트워크 오류가 발생하면, **When** 등록 실패 시, **Then** 에러 메시지가 표시되고 입력 내용이 유지된다

---

### User Story 4 - 비밀글 피드백의 댓글 비밀 처리 (Priority: P2)

사용자가 비밀글로 작성한 피드백에 관리자가 댓글을 달면, 해당 댓글도 대시보드 페이지에서 비밀글로 표시된다. 관리자 페이지에서는 모든 피드백과 댓글 내용을 확인할 수 있다.

**Why this priority**: 사용자 프라이버시 보호를 위한 중요한 기능이지만, 기본 CRUD 이후 구현 가능

**Independent Test**: 비밀글 피드백에 관리자가 댓글을 달고, 대시보드에서 해당 댓글이 비밀 처리되어 보이는지 확인

**Acceptance Scenarios**:

1. **Given** 비밀글 피드백에 관리자가 댓글을 작성했을 때, **When** 피드백 작성자가 대시보드에서 확인하면, **Then** 자신의 피드백과 관리자 댓글을 모두 볼 수 있다
2. **Given** 비밀글 피드백에 관리자가 댓글을 작성했을 때, **When** 다른 사용자가 대시보드에서 확인하면, **Then** 해당 피드백과 댓글 내용이 "비밀글입니다"로 표시된다
3. **Given** 비밀글이 아닌 피드백에 관리자가 댓글을 작성했을 때, **When** 모든 사용자가 대시보드에서 확인하면, **Then** 피드백과 댓글이 정상적으로 표시된다

---

### Edge Cases

- 피드백이 소프트 삭제(deleted_at)된 경우 → 삭제된 피드백은 목록에서 제외 (deleted_at IS NULL 필터링)
- 댓글 작성 중 동시에 다른 관리자가 같은 피드백에 댓글을 작성한 경우 → 등록 후 최신 목록을 다시 조회하여 갱신
- 피드백 작성자 계정이 삭제된 경우 → "탈퇴한 사용자"로 표시
- 매우 긴 댓글 내용 → 기존 DB 제약조건(char_length > 0) 활용, 프론트엔드에서 적절한 최대 길이 제한
- 페이지네이션에서 존재하지 않는 페이지 요청 → 빈 목록 표시

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 모든 스킬의 피드백을 테이블 형태로 표시해야 한다 (스킬명, 작성자, 평점, 코멘트, 비밀글 여부, 작성일, 댓글 수 포함)
- **FR-002**: 피드백 목록은 최신순(작성일 기준 내림차순)으로 정렬되어야 한다
- **FR-003**: 피드백 목록에 페이지네이션이 적용되어야 한다 (페이지당 10건)
- **FR-004**: 관리자가 피드백 행을 클릭하면 아코디언이 펼쳐져 댓글 목록과 댓글 작성 폼이 표시되어야 한다
- **FR-005**: 댓글 목록은 최신순으로 표시되어야 한다
- **FR-006**: 관리자는 아코디언 내 댓글 작성 폼을 통해 댓글을 등록할 수 있어야 한다
- **FR-007**: 빈 내용의 댓글은 등록할 수 없어야 한다 (유효성 검증)
- **FR-008**: 비밀글 피드백에 달린 관리자 댓글은 대시보드에서 비밀글과 동일하게 처리되어야 한다 (작성자 본인과 관리자만 내용 확인 가능)
- **FR-009**: 관리자 페이지에서는 비밀글 여부와 관계없이 모든 피드백과 댓글 내용을 확인할 수 있어야 한다
- **FR-010**: 소프트 삭제된 피드백(deleted_at이 있는 피드백)은 목록에서 제외되어야 한다

### Key Entities

- **피드백(skill_feedback_logs)**: 사용자가 스킬에 대해 남긴 평점과 코멘트. 비밀글 여부(is_secret) 속성을 가지며, 소프트 삭제(deleted_at) 지원. 스킬 및 사용자와 연관됨
- **댓글(feedback_replies)**: 피드백에 대한 응답. 피드백과 N:1 관계이며, 작성자(관리자 또는 사용자)와 연관됨. 내용은 빈 값 불가
- **스킬(skills)**: 피드백의 대상이 되는 엔티티. 피드백 테이블에 스킬명 표시를 위해 조인 필요
- **프로필(profiles)**: 피드백/댓글 작성자 정보 표시를 위해 조인 필요

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 관리자가 피드백 관리 페이지에서 3초 이내에 피드백 목록을 확인할 수 있다
- **SC-002**: 관리자가 피드백에 대한 댓글을 30초 이내에 작성하고 등록할 수 있다
- **SC-003**: 비밀글 피드백의 댓글이 대시보드에서 비밀 처리되어 작성자 외 사용자에게 내용이 노출되지 않는다
- **SC-004**: 페이지네이션이 정상 동작하여 전체 피드백을 페이지 단위로 탐색할 수 있다

## Assumptions

- 기존 `skill_feedback_logs` 및 `feedback_replies` 테이블 스키마를 변경 없이 사용한다
- 기존 `/admin/feedbacks` 페이지 라우트와 기본 구조(SupabaseAdminRepository, GetFeedbacksUseCase)를 확장한다
- 페이지당 피드백 수는 10건으로 설정한다 (기존 코드와 동일)
- 관리자 권한 검증은 기존 어드민 레이아웃의 인증 로직을 따른다
- 대시보드에서 비밀글 댓글 처리는 기존 is_secret 필드 기반 로직을 활용한다
