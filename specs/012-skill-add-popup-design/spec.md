# Feature Specification: 스킬 추가 팝업 디자인 리뉴얼

**Feature Branch**: `012-skill-add-popup-design`
**Created**: 2026-03-04
**Status**: Draft
**Input**: User description: "스킬 추가 팝업의 레이아웃구조와 스타일을 stitch-html/admin-add-skill.html 참고하여 리뉴얼하고, 상세설명에 마크다운 파일 업로드 시 미리보기 기능 추가"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 마크다운 설명 파일 업로드 및 미리보기 (Priority: P1)

관리자가 스킬의 상세 설명을 작성할 때, 마크다운(.md) 파일을 업로드하면 모달 내에서 렌더링된 미리보기를 바로 확인할 수 있다. 이를 통해 등록 전에 최종 결과물을 시각적으로 검증할 수 있다.

**Why this priority**: 마크다운 미리보기는 이번 기능의 핵심 신규 기능이며, 관리자가 스킬 설명의 품질을 등록 전에 확인할 수 있는 유일한 방법이다. 현재는 파일만 업로드되고 내용을 확인할 수 없어 오류가 포함된 설명이 그대로 게시될 위험이 있다.

**Independent Test**: 마크다운 파일을 업로드하고 렌더링된 미리보기가 올바르게 표시되는지 확인하면 독립적으로 테스트 가능하다.

**Acceptance Scenarios**:

1. **Given** 스킬 추가 모달이 열려있고 상세 설명 영역이 비어있을 때, **When** 관리자가 .md 파일을 업로드하면, **Then** 해당 마크다운 내용이 서식이 적용된 형태(제목, 목록, 코드 블록, 링크 등)로 미리보기 영역에 렌더링된다.
2. **Given** 마크다운 미리보기가 표시된 상태에서, **When** 관리자가 다른 .md 파일을 업로드하면, **Then** 미리보기가 새 파일의 내용으로 교체된다.
3. **Given** 마크다운 미리보기가 표시된 상태에서, **When** 관리자가 업로드된 파일을 제거하면, **Then** 미리보기 영역이 초기 상태(업로드 안내)로 돌아간다.
4. **Given** 관리자가 .md가 아닌 파일을 업로드하려 할 때, **When** 파일 선택을 시도하면, **Then** .md 파일만 허용된다는 안내가 표시된다.

### User Story 2 - 리뉴얼된 2패널 레이아웃의 스킬 등록 경험 (Priority: P2)

관리자가 스킬을 등록할 때, 좌측에는 주요 콘텐츠(아이콘, 제목, 상세 설명)를, 우측 사이드바에는 메타데이터(카테고리, 템플릿 파일, 저장 버튼)를 배치한 2패널 레이아웃으로 정보를 효율적으로 입력할 수 있다.

**Why this priority**: 레이아웃 리뉴얼은 기존 기능의 시각적·구조적 개선으로, 새로운 기능(P1)보다는 후순위이지만 사용성 향상에 큰 영향을 미친다.

**Independent Test**: 모달을 열어 2패널 구조가 올바르게 표시되고, 각 영역의 폼 요소가 정상 작동하는지 확인하면 독립적으로 테스트 가능하다.

**Acceptance Scenarios**:

1. **Given** 관리자가 스킬 관리 페이지에서 "새 스킬 등록" 버튼을 클릭할 때, **When** 모달이 열리면, **Then** 좌측 콘텐츠 영역과 우측 사이드바가 구분된 2패널 레이아웃이 표시된다.
2. **Given** 2패널 모달이 열린 상태에서, **When** 좌측 영역에서 아이콘 선택, 제목 입력, 상세 설명 파일 업로드를 수행하면, **Then** 각 입력이 올바르게 반영되고 우측 사이드바의 카테고리·템플릿 입력과 독립적으로 동작한다.
3. **Given** 모바일 또는 좁은 화면에서 모달이 열릴 때, **When** 화면 너비가 작으면, **Then** 2패널이 세로 방향으로 쌓여 단일 컬럼으로 표시된다.

### User Story 3 - 마크다운 미리보기의 다양한 서식 지원 (Priority: P3)

관리자가 업로드한 마크다운 파일에 다양한 서식(제목, 강조, 코드 블록, 테이블, 이미지, 링크 등)이 포함되어 있을 때, 미리보기가 이를 충실하게 렌더링한다.

**Why this priority**: 기본 미리보기(P1)가 동작한 후, 다양한 마크다운 문법에 대한 완성도를 높이는 보완 기능이다.

**Independent Test**: 다양한 마크다운 문법이 포함된 테스트 파일을 업로드하여 각 서식 요소가 올바르게 렌더링되는지 확인하면 독립적으로 테스트 가능하다.

**Acceptance Scenarios**:

1. **Given** 마크다운 파일에 제목(h1~h6), 볼드, 이탤릭, 코드 블록이 포함되어 있을 때, **When** 파일을 업로드하면, **Then** 각 서식 요소가 시각적으로 구분되어 렌더링된다.
2. **Given** 마크다운 파일에 테이블이 포함되어 있을 때, **When** 파일을 업로드하면, **Then** 테이블이 정렬된 행/열 구조로 표시된다.
3. **Given** 마크다운 파일의 내용이 미리보기 영역 높이를 초과할 때, **When** 미리보기가 렌더링되면, **Then** 미리보기 영역 내에서 스크롤이 가능하다.

---

### Edge Cases

- 관리자가 0바이트(빈) .md 파일을 업로드하면 어떻게 처리하는가? → 빈 파일 안내 메시지를 표시한다.
- 관리자가 1MB를 초과하는 대용량 .md 파일을 업로드하면 어떻게 처리하는가? → 파일 크기 제한 초과 안내를 표시하고 업로드를 거부한다.
- 마크다운 파일 내에 악성 스크립트(XSS)가 포함된 경우 어떻게 처리하는가? → HTML/스크립트 태그가 렌더링되지 않도록 sanitize 처리한다.
- 브라우저가 FileReader API를 지원하지 않는 환경에서는 어떻게 처리하는가? → 파일 업로드는 허용하되 미리보기 불가 안내를 표시한다.
- 마크다운 파일의 인코딩이 UTF-8이 아닌 경우 어떻게 처리하는가? → UTF-8로 읽기를 시도하고 깨진 문자가 있으면 인코딩 확인 안내를 표시한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 상세 설명 영역에서 .md 파일 업로드 기능을 제공해야 한다.
- **FR-002**: 시스템은 업로드된 .md 파일의 내용을 읽어 서식이 적용된 미리보기를 모달 내에 표시해야 한다.
- **FR-003**: 미리보기는 제목(h1~h6), 볼드, 이탤릭, 취소선, 순서/비순서 목록, 코드 블록(인라인/블록), 테이블, 링크, 이미지를 지원해야 한다.
- **FR-004**: 시스템은 마크다운 내 HTML 태그 및 스크립트를 sanitize하여 XSS 공격을 방지해야 한다.
- **FR-005**: 시스템은 업로드된 파일을 다른 파일로 교체하거나 제거할 수 있어야 한다.
- **FR-006**: 시스템은 .md 파일만 허용하고, 다른 형식의 파일 선택 시 적절한 안내를 표시해야 한다.
- **FR-007**: 시스템은 1MB 이하의 .md 파일만 허용해야 한다.
- **FR-008**: 스킬 추가 모달은 좌측 콘텐츠 영역(아이콘 선택, 제목 입력, 상세 설명)과 우측 사이드바(카테고리, 템플릿 파일, 저장/취소 버튼)로 구성된 2패널 레이아웃이어야 한다.
- **FR-009**: 모달은 모바일 등 좁은 화면에서 세로 단일 컬럼으로 자동 전환되어야 한다.
- **FR-010**: 기존 스킬 추가 모달의 모든 기능(이모지 아이콘 선택, 카테고리, 제목, 설명, 템플릿 파일 업로드, 공개 여부, 임시저장, 닫기 확인)이 리뉴얼 후에도 유지되어야 한다.
- **FR-011**: 빈(0바이트) .md 파일 업로드 시 빈 파일 안내 메시지를 표시해야 한다.

### Key Entities

- **스킬(Skill)**: 관리자가 등록하는 AI 자동화 스킬. 아이콘, 제목, 카테고리, 상세 설명(마크다운), 템플릿 파일, 공개 상태를 포함한다.
- **설명 마크다운 파일(Description Markdown File)**: 스킬의 상세 사용 가이드를 담은 .md 파일. 업로드 시 모달 내에서 렌더링 미리보기를 제공한다.
- **템플릿 파일(Template File)**: 스킬에 첨부되는 보조 파일(.md, .zip). 상세 설명 마크다운과는 별도로 관리된다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 관리자가 .md 파일을 업로드한 후 3초 이내에 서식이 적용된 미리보기가 표시된다.
- **SC-002**: 미리보기에서 제목, 목록, 코드 블록, 테이블, 링크 등 주요 마크다운 서식 5종 이상이 올바르게 렌더링된다.
- **SC-003**: 모달의 2패널 레이아웃이 데스크톱(1024px 이상)에서 좌우 분할, 모바일(768px 미만)에서 세로 단일 컬럼으로 표시된다.
- **SC-004**: 기존 스킬 등록 기능(아이콘, 제목, 카테고리, 템플릿, 저장, 임시저장, 닫기 확인) 100%가 리뉴얼 후에도 정상 동작한다.
- **SC-005**: 마크다운 내 스크립트/HTML 삽입 시도 시 100% sanitize되어 렌더링에 반영되지 않는다.

## Assumptions

- 관리자는 이미 인증 및 권한 검증이 완료된 상태에서 스킬 추가 모달에 접근한다.
- 마크다운 파일은 클라이언트에서 읽어 미리보기를 생성하며, 서버에 저장하는 것은 스킬 저장 시점에 수행된다.
- .md 파일의 최대 크기 제한은 기존 정책(1MB)을 따른다.
- 마크다운 렌더링은 React Markdown + remark/rehype 플러그인을 사용한다(사용자 지정 요구사항).
- 레이아웃 스타일은 `stitch-html/admin-add-skill.html`의 디자인을 기준으로 하되, 기존 프로젝트의 디자인 시스템(Shadcn UI, Tailwind CSS)과 일관성을 유지한다.

## Scope & Boundaries

### In Scope
- 마크다운 파일 업로드 및 클라이언트 사이드 미리보기 렌더링
- 2패널 레이아웃으로 모달 구조 리뉴얼
- 기존 폼 기능 유지 및 통합
- 마크다운 XSS 방지를 위한 sanitize 처리

### Out of Scope
- 마크다운 직접 편집기(WYSIWYG/에디터) 기능
- 마크다운 파일의 서버 사이드 렌더링
- 스킬 수정/삭제 기능 (별도 기능으로 구현)
- 이미지 호스팅 또는 마크다운 내 이미지 업로드 기능
