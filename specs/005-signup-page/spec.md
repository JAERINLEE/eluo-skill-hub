# Feature Specification: 회원가입 페이지

**Feature Branch**: `005-signup-page`
**Created**: 2026-03-03
**Status**: Draft
**Input**: User description: "회원가입 페이지 및 회원가입 기능 구현"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 이메일/비밀번호로 회원가입 (Priority: P1)

eluocnc.com 소속 직원이 회원가입 페이지(`/signup`)에서 사내 이메일과 비밀번호를 입력하여 계정을 생성한다. 시스템은 입력값을 검증한 후 해당 이메일로 8자리 인증코드를 발송한다.

**Why this priority**: 회원가입의 핵심 흐름으로, 이메일/비밀번호 입력 및 유효성 검증 없이는 어떤 후속 단계도 진행할 수 없다.

**Independent Test**: 회원가입 폼에 유효한 이메일과 비밀번호를 입력하고 제출했을 때, 인증코드 발송까지 정상 동작하는지 확인 가능

**Acceptance Scenarios**:

1. **Given** 사용자가 `/signup` 페이지에 접속한 상태, **When** `user@eluocnc.com` 형식의 이메일과 8자 이상의 비밀번호를 입력하고 회원가입 버튼을 누르면, **Then** 입력한 이메일로 8자리 인증코드가 발송되고 인증코드 입력 화면으로 전환된다
2. **Given** 사용자가 회원가입 폼을 작성 중인 상태, **When** `@eluocnc.com`이 아닌 이메일(예: `user@gmail.com`)을 입력하면, **Then** "eluocnc.com 이메일만 사용할 수 있습니다" 오류 메시지가 표시된다
3. **Given** 사용자가 회원가입 폼을 작성 중인 상태, **When** 8자 미만의 비밀번호를 입력하면, **Then** "비밀번호는 최소 8자 이상이어야 합니다" 오류 메시지가 표시된다
4. **Given** 사용자가 회원가입 폼을 작성 중인 상태, **When** 이메일 또는 비밀번호 필드가 비어있는 상태로 회원가입 버튼을 누르면, **Then** 비어있는 필드에 대한 필수 입력 안내 메시지가 표시된다

---

### User Story 2 - 인증코드 확인으로 회원가입 완료 (Priority: P1)

사용자가 이메일로 받은 8자리 인증코드를 입력하여 회원가입을 최종 완료한다. 인증이 성공하면 profiles 테이블에 회원 정보가 추가된다.

**Why this priority**: 인증코드 확인은 회원가입 완료의 필수 단계로, 이 단계 없이는 계정이 활성화되지 않는다.

**Independent Test**: 인증코드 입력 화면에서 올바른 8자리 코드를 입력했을 때 회원가입이 완료되고 profiles에 데이터가 생성되는지 확인 가능

**Acceptance Scenarios**:

1. **Given** 인증코드가 이메일로 발송된 상태, **When** 올바른 8자리 인증코드를 입력하고 확인 버튼을 누르면, **Then** 회원가입이 완료되고 profiles에 회원 정보(이메일, 기본 역할)가 저장되며 대시보드(`/dashboard`)로 이동한다
2. **Given** 인증코드 입력 화면인 상태, **When** 잘못된 인증코드를 입력하면, **Then** "인증코드가 올바르지 않습니다" 오류 메시지가 표시되고 재입력할 수 있다
3. **Given** 인증코드 입력 화면인 상태, **When** 인증코드 유효시간(10분)이 경과한 후 코드를 입력하면, **Then** "인증코드가 만료되었습니다. 다시 시도해주세요" 메시지가 표시된다

---

### User Story 3 - 인증코드 재발송 (Priority: P2)

사용자가 인증코드를 받지 못했거나 유효시간이 경과한 경우, 인증코드를 재발송 받을 수 있다.

**Why this priority**: 인증코드 미수신이나 만료 시 회원가입 흐름이 중단되지 않도록 하는 보조 기능이다.

**Independent Test**: 인증코드 입력 화면에서 재발송 버튼을 눌렀을 때 새로운 인증코드가 발송되는지 확인 가능

**Acceptance Scenarios**:

1. **Given** 인증코드 입력 화면인 상태, **When** "인증코드 재발송" 버튼을 누르면, **Then** 새로운 8자리 인증코드가 이메일로 발송되고 "인증코드가 재발송되었습니다" 안내 메시지가 표시된다
2. **Given** 인증코드 재발송 버튼을 누른 직후, **When** 연속으로 재발송 버튼을 누르면, **Then** 최소 60초 간격으로만 재발송이 가능하며 남은 대기 시간이 표시된다
3. **Given** 이메일 발송 횟수 제한에 도달한 상태, **When** 재발송 버튼을 누르면, **Then** "이메일 발송 횟수 제한에 도달했습니다. 잠시 후 다시 시도해주세요" 메시지가 표시되고 재발송 버튼이 비활성화된다

---

### User Story 4 - 기존 회원 안내 (Priority: P3)

이미 가입된 이메일로 회원가입을 시도하는 사용자에게 적절한 안내를 제공한다.

**Why this priority**: 중복 가입 방지 및 기존 회원의 원활한 서비스 이용을 위한 안내 기능이다.

**Independent Test**: 이미 가입된 이메일로 회원가입을 시도했을 때 적절한 안내 메시지가 표시되는지 확인 가능

**Acceptance Scenarios**:

1. **Given** `user@eluocnc.com`으로 이미 가입된 상태, **When** 동일한 이메일로 회원가입을 시도하면, **Then** "이미 가입된 이메일입니다" 메시지와 함께 로그인 페이지로 이동할 수 있는 링크가 표시된다

---

### Edge Cases

- 인증코드 입력 중 브라우저를 닫거나 페이지를 떠난 경우: 재접속 후 동일 이메일로 회원가입을 시도하면, 미인증(pending) 계정임을 감지하여 OTP 입력 화면으로 자동 전환된다. 사용자는 이미 받은 코드를 입력하거나 재발송 버튼을 이용할 수 있다
- 이메일 발송 rate limit 초과 시: "이메일 발송 횟수 제한에 도달했습니다. 잠시 후 다시 시도해주세요" 메시지를 표시하고 재발송 버튼을 비활성화한다
- 네트워크 오류로 인증코드 발송에 실패한 경우: "인증코드 발송에 실패했습니다. 다시 시도해주세요" 메시지를 표시한다
- 비밀번호에 특수문자, 한글, 공백 등이 포함된 경우: 8자 이상이면 모든 문자 조합을 허용한다
- 이메일 대소문자 혼용 입력(예: `User@EluoCNC.com`): 대소문자 구분 없이 동일 이메일로 처리한다

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 `/signup` 경로에 회원가입 페이지를 제공해야 한다
- **FR-002**: 회원가입 폼은 이메일 입력 필드, 비밀번호 입력 필드, 회원가입 버튼으로 구성되어야 한다
- **FR-003**: 시스템은 이메일 도메인이 `eluocnc.com`인 경우에만 입력을 허용해야 한다
- **FR-004**: 시스템은 비밀번호가 최소 8자 이상인 경우에만 입력을 허용해야 한다
- **FR-005**: 유효성 검증 통과 시, 시스템은 입력된 이메일로 8자리 인증코드를 발송해야 한다
- **FR-006**: 인증코드는 숫자로 구성된 8자리이며, 발송 후 10분간 유효해야 한다
- **FR-007**: 사용자가 올바른 인증코드를 입력하면 회원가입이 완료되고 profiles 테이블에 회원 정보가 추가되어야 한다
- **FR-008**: 회원가입 시 기본 역할은 "user"로 설정되어야 한다
- **FR-009**: 이미 가입된 이메일로 회원가입을 시도할 경우, 미인증(이메일 미확인) 상태이면 OTP 입력 화면으로 전환하고, 이미 인증 완료된 계정이면 "이미 가입된 이메일입니다" 안내 메시지와 로그인 링크를 표시해야 한다
- **FR-010**: 인증코드 재발송 기능을 제공하며, 최소 60초 간격 제한을 적용해야 한다
- **FR-010a**: 이메일 발송 rate limit 초과 시 "이메일 발송 횟수 제한에 도달했습니다. 잠시 후 다시 시도해주세요" 메시지를 표시하고 재발송 버튼을 비활성화해야 한다
- **FR-011**: 회원가입 완료 후 대시보드(`/dashboard`)로 자동 이동해야 한다
- **FR-012**: 회원가입 페이지에서 로그인 페이지로 이동할 수 있는 링크를 제공해야 한다

### Key Entities

- **사용자 계정(User Account)**: 이메일(eluocnc.com 도메인), 비밀번호(8자 이상)로 구성된 인증 정보. 인증 완료 시 활성화됨
- **프로필(Profile)**: 사용자 계정에 연결된 회원 정보. id(사용자 고유 식별자), email, role_id(기본값: user), created_at을 포함
- **인증코드(Verification Code)**: 회원가입 시 이메일로 발송되는 8자리 숫자 코드. 10분간 유효하며 회원가입 완료에 필수

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 회원가입 전체 과정(폼 입력 ~ 인증코드 확인)을 3분 이내에 완료할 수 있다
- **SC-002**: eluocnc.com 외 도메인 이메일 입력 시 100% 차단된다
- **SC-003**: 인증코드 발송 후 30초 이내에 사용자 이메일로 도달한다
- **SC-004**: 유효한 인증코드 입력 시 회원가입 성공률이 99% 이상이다
- **SC-005**: 회원가입 완료 후 profiles에 회원 정보가 즉시 생성된다
- **SC-006**: 90% 이상의 신규 사용자가 별도 도움 없이 회원가입을 완료할 수 있다

## Clarifications

### Session 2026-03-03

- Q: 회원가입 후 창을 닫고 재접속 시 처리 방법 → A: 동일 이메일로 재시도 시 미인증(pending) 계정 감지 후 OTP 입력 화면으로 자동 전환. 기존 코드 입력 또는 재발송 가능
- Q: 이메일 rate limit 초과 시 처리 → A: "이메일 발송 횟수 제한에 도달했습니다. 잠시 후 다시 시도해주세요" 메시지 표시 + 재발송 버튼 비활성화

## Assumptions

- 이메일 발송 인프라는 이미 구축되어 있거나 프로젝트에서 사용하는 인증 서비스에서 제공한다
- 비밀번호 확인(재입력) 필드는 요구사항에 명시되지 않아 포함하지 않는다 (필요시 추가 가능)
- 인증코드 유효시간은 일반적인 표준인 10분으로 설정한다
- 인증코드 재발송 시 이전 코드는 무효화된다
- 회원가입 완료 후 자동 로그인은 포함하지 않으며, 로그인 페이지로 이동한다
- 이메일 대소문자는 구분하지 않는다 (예: `User@EluoCNC.com` = `user@eluocnc.com`)
