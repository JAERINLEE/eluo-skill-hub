# Implementation Plan

- [x] 1. 미들웨어에 비인증 사용자 보호 경로 가드 추가
- [x] 1.1 비인증 사용자의 루트 페이지 접근 차단 로직 구현
  - 기존 미들웨어에서 `getUser()` 결과를 활용하여, 인증되지 않은 사용자가 보호 경로(`/`)에 접근할 때 로그인 페이지로 리다이렉트하는 조건 분기를 추가한다
  - auth 페이지 경로 목록(`/login`, `/signup`)을 상수로 관리하여 유지보수성을 확보한다
  - 기존 인증 사용자의 auth 페이지 접근 시 루트 페이지로 리다이렉트하는 동작이 유지되는지 확인한다
  - Supabase Auth 세션 기반으로 인증 상태를 판별하며, `getSession()` 대신 `getUser()`만 사용한다
  - _Requirements: 1.1, 1.2, 1.3, 1.4_

- [x] 2. 서버 컴포넌트에서 인증 정보를 조회하여 클라이언트에 전달
- [x] 2.1 루트 페이지 서버 컴포넌트를 async로 전환하고 사용자 이메일을 조회하여 전달
  - 기존 동기 함수 컴포넌트를 async 서버 컴포넌트로 변환하여 Supabase Auth에서 사용자 정보를 조회한다
  - 사용자 이메일만 추출하여 DashboardShell 컴포넌트에 props로 전달한다
  - 토큰, 비밀번호 등 민감한 데이터는 클라이언트에 노출하지 않는다
  - _Requirements: 5.1, 5.2_

- [x] 2.2 DashboardShell 컴포넌트가 userEmail props를 수신하여 AppHeader에 전달하도록 수정
  - DashboardShell의 props 인터페이스에 userEmail 프로퍼티를 추가한다
  - 수신한 userEmail을 AppHeader 컴포넌트에 그대로 전달한다
  - 기존 대시보드 상태 관리 훅과의 관계에 영향이 없도록 한다
  - _Requirements: 5.1_

- [x] 3. 로그아웃 Server Action에 에러 처리 추가
- [x] 3.1 (P) 기존 logoutAction을 확장하여 성공/실패 결과를 반환하도록 수정
  - 기존 logoutAction에 try-catch 기반 에러 처리를 추가하여 LogoutResult 타입(성공/에러 판별 합집합)을 반환하도록 한다
  - AuthRepository의 signOut 메서드를 호출하여 Supabase Auth 세션을 종료한다
  - 성공 시 로그인 페이지로 리다이렉트하며, redirect 호출은 try-catch 블록 외부에서 실행하여 Next.js의 NEXT_REDIRECT 에러 충돌을 방지한다
  - 실패 시 에러 메시지를 포함한 결과 객체를 반환하여 클라이언트에서 오류를 표시할 수 있게 한다
  - 로그아웃 완료 시 클라이언트 측 인증 상태가 초기화되도록 보장한다
  - _Requirements: 4.1, 4.2, 4.3, 4.4_

- [x] 4. 사용자 프로필 팝오버 컴포넌트 생성
- [x] 4.1 (P) 프로필 아이콘 클릭 시 계정 정보와 로그아웃 버튼을 표시하는 팝오버 구현
  - Radix UI Popover를 활용하여 프로필 아이콘 클릭 시 팝오버를 열고, 외부 클릭 시 자동으로 닫히도록 한다
  - 팝오버 내부에 로그인한 사용자의 이메일 주소를 표시한다
  - 로그아웃 버튼을 포함하며, 클릭 시 logoutAction을 호출한다
  - 로그아웃 진행 중에는 로딩 상태를 표시하고 버튼을 비활성화하여 중복 호출을 방지한다
  - 로그아웃 실패 시 팝오버 내부에 오류 메시지를 표시하고, 팝오버가 닫히면 에러 상태를 초기화한다
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 4.1, 4.2, 4.3, 4.4_

- [x] 5. AppHeader에 인증 상태 기반 프로필 아이콘 활성화 및 팝오버 연결
- [x] 5.1 AppHeader의 프로필 영역을 인증 상태에 따라 활성화하고 팝오버를 렌더링하도록 수정
  - AppHeader의 props 인터페이스에 userEmail 프로퍼티를 추가한다
  - userEmail이 존재하면 프로필 아이콘을 클릭 가능한 활성 상태로 전환하고, 인증 상태를 시각적으로 구분할 수 있는 스타일(강조 색상 테두리 등)을 적용한다
  - 기존 비대화형 프로필 아이콘 영역을 UserProfilePopover 컴포넌트로 교체하여 연결한다
  - 프로필 영역의 로딩 상태를 처리한다
  - _Requirements: 2.1, 2.2, 5.1, 5.3_

- [x] 6. 통합 검증 및 테스트
- [x] 6.1 미들웨어 인증 가드 통합 테스트 작성
  - 비인증 사용자가 루트 페이지에 접근할 때 로그인 페이지로 리다이렉트되는지 검증한다
  - 인증 사용자가 루트 페이지에 정상적으로 접근할 수 있는지 검증한다
  - 인증 사용자가 로그인/회원가입 페이지에 접근할 때 루트 페이지로 리다이렉트되는지 검증한다
  - _Requirements: 1.1, 1.2, 1.3_

- [x] 6.2 로그아웃 Server Action 단위 테스트 작성
  - 로그아웃 성공 시 세션 종료 후 로그인 페이지로 리다이렉트되는지 검증한다
  - 로그아웃 실패 시 에러 메시지를 포함한 결과 객체가 반환되는지 검증한다
  - _Requirements: 4.1, 4.2, 4.4_

- [x] 6.3 UserProfilePopover 컴포넌트 단위 테스트 작성
  - 이메일 주소가 올바르게 표시되는지 검증한다
  - 로그아웃 버튼이 렌더링되고 클릭 시 logoutAction이 호출되는지 검증한다
  - 로그아웃 실패 시 오류 메시지가 팝오버 내부에 표시되는지 검증한다
  - _Requirements: 3.2, 3.3, 4.4_

- [x]* 6.4 AppHeader 인증 상태 렌더링 테스트 작성
  - userEmail props 유무에 따라 프로필 아이콘의 활성/비활성 상태가 올바르게 렌더링되는지 검증한다
  - 인증 상태의 시각적 구분 스타일이 적용되는지 검증한다
  - _Requirements: 2.1, 2.2_

- [x]* 6.5 서버-클라이언트 인증 정보 전달 흐름 통합 테스트 작성
  - 서버 컴포넌트에서 조회한 사용자 이메일이 DashboardShell을 거쳐 AppHeader까지 정상적으로 전달되는지 검증한다
  - 민감 데이터(토큰, 비밀번호)가 클라이언트에 노출되지 않는지 검증한다
  - _Requirements: 5.1, 5.2_
